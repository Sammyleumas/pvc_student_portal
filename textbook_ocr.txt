PREFACE
Welcome to the official textbook for the Professional Vibe Coding and AI Development (PVC-AID) programme offered by SL-TECHCO Academy. This book is designed to be your complete companion throughout the course — from your very first AI-generated webpage all the way to deploying a full SaaS product and building a freelancing career.
Vibe Coding describes a modern, intuitive approach to software development where you work alongside artificial intelligence rather than writing every line of code yourself. Instead of memorising syntax, you learn to communicate effectively with AI tools, guide them with well-crafted prompts, evaluate their output critically, and assemble production-quality software faster than ever before.
This textbook covers 14 modules plus a capstone project.

MODULE 1: INTRODUCTION TO VIBE CODING
- Learning Objectives: Understand Vibe Coding and its origin, compare traditional coding to AI-assisted development, identify benefits and limitations, understand the role of prompt engineering, set up AI workspace, create first AI webpage.
- Concepts:
  * Coined by Andrej Karpathy in 2025: "I just vibe code these days - I fully give in to the vibes, embrace not fully understanding all the code, and it mostly works."
  * Vibe Coding still requires technical knowledge. You must know what good code looks like, how to test it, and how to fix it when it breaks.
  * Traditional vs Vibe Coding: Traditional learning curve is steep (years of study), Vibe is gentle (natural language first). Traditional speed to prototype is weeks to months, Vibe is hours to days. Traditional debugging is manual, Vibe uses AI assistance. Cost of errors is lower in Vibe.
  * Benefits of Vibe Coding: speed, accessibility (no formal CS degree needed), rapid exploration, learn by watching AI, focus on business logic and UX rather than syntax, smaller teams can build larger products.
  * Limitations of Vibe Coding: hallucinations (AI generates buggy code that looks correct), security gaps, over-reliance (cannot debug or improve if you don't understand), context limits, unpredictability, IP debates.
  * AI Workspace: Browser (Chrome/Firefox), AI Chat Tools (ChatGPT, Claude, Gemini), Code Editor (VS Code), Extensions (Prettier, Live Server, GitLens), Version Control (Git/GitHub), Cursor AI.
- Module 1 Assignment: Create a complete personal profile webpage using AI assistance. Your page must include: a professional header with your name and photo placeholder, an About Me section, a Skills section, a Projects section (at least 2 imaginary or real projects), and a styled Contact form. Save as a single HTML file.

MODULE 2: AI TOOLS FOR VIBE CODING
- Learning Objectives: Understand strengths of major AI tools, compare outputs, choose the right AI tool, generate and evaluate code.
- Tools Overview:
  * ChatGPT (OpenAI): Most widely used. GPT-4o is excellent for code generation, explanation, debugging, and has strong plugins. Best for: general coding, debugging, content creation, code explanation.
  * Claude (Anthropic): Known for long document handling, nuanced reasoning, and writing quality. Claude 3.5 Sonnet excels at complex multi-file coding tasks and understanding large codebases. Best for: large project coding, technical writing, code review.
  * Gemini AI (Google): Flagship AI with strong Google Workspace integration. Gemini 1.5 Pro has the largest context window of any consumer model. Best for: Google-ecosystem tasks, research, multimodal inputs.
  * DeepSeek AI: Chinese AI that rivals GPT-4 at coding tasks. DeepSeek Coder is specifically optimized for programming and is free to use. Best for: code generation, algorithmic problems, free alternative.
  * Grok AI (xAI): Elon Musk's AI integrated with X. Has real-time internet access, humorous/direct communication. Best for: real-time info, brainstorming, social media.
  * Microsoft Copilot: Powered by GPT-4, integrated with Microsoft 365. Best for: office document automation, business writing, Bing search.
  * Perplexity AI: Research-focused AI that cites sources. Best for: research, fact-checking, finding documentation, current events.
  * Cursor AI: VS Code fork with built-in AI. Can highlight code and ask questions, edit in-context. Best for: in-editor coding assistance, refactoring, whole-project context.
  * GitHub Copilot: AI pair programmer, VS Code extension. Best for: autocomplete, boilerplate code, in-editor suggestions.

MODULE 3: PROMPT ENGINEERING FOR DEVELOPERS
- Learning Objectives: Understand the anatomy of a high-quality prompt, apply context, chain-of-thought, and few-shot prompting, use iterative prompting, debug through prompts, build library of templates.
- Anatomy of a Good Prompt:
  1. Role Assignment: Tell AI who it is ("Act as a senior React developer...").
  2. Task Description: State the task in one clear sentence ("Create a responsive product card component...").
  3. Context: Provide relevant background ("This is for an e-commerce site targeting Nigerian shoppers...").
  4. Constraints: Define what the AI must NOT do ("Do not use any CSS frameworks. Write plain CSS only.").
  5. Output Format: Specify output format ("Return only the HTML and CSS. Do not add explanations.").
  6. Examples (Optional): Show reference/example ("Like this: [code]").
  7. Quality Bar: State the standard ("The code must be clean, commented, and follow BEM...").
- Prompting Techniques:
  * Context Prompting: Provide rich background information before asking.
  * Chain-of-Thought Prompting: Ask AI to reason step-by-step before generating code ("Before writing any code, think step by step...").
  * Few-Shot Prompting: Provide 2-3 examples of the input-output pattern you want.
  * Iterative Prompting: Start with a rough version and refine through follow-up prompts.
  * Debugging through prompts: Paste code with bug and a clear description of the error.
- Reusable Templates: New Component, Bug Fix, Code Review, Feature Addition, API Integration.
- Module 3 Assignment: Develop a personal library of 20 reusable coding prompt templates. Your templates must cover at least: 5 for frontend development, 3 for backend/database tasks, 3 for debugging, 3 for code review/refactoring, 3 for mobile app development, and 3 of your own design.

MODULE 4: WEB DEVELOPMENT WITH AI
- Learning Objectives: Understand HTML, CSS, JS fundamentals; use AI to build responsive websites efficiently; create landing pages/portfolios; understand responsive design; build and deploy complete business website.
- Concepts:
  * HTML: HyperText Markup Language, the skeleton of every webpage.
  * CSS: Cascading Style Sheets, controls how elements look (colors, fonts, spacing, layout, animations). Use Flexbox and Grid.
  * JS: JavaScript, adds interactivity (responding to clicks, fetching data, animating elements, validating forms).
  * Responsive Design: Must look good on all screen sizes. Over 70% of traffic in Nigeria comes from mobile. Use relative units (%, em, rem, vw, vh), Flexbox/Grid, media queries (@media), viewport meta tag, design mobile-first, and test on real devices.
  * AI-Assisted Frontend Workflow: 1. Plan site (wireframe), 2. Write detailed prompt, 3. Generate initial HTML/CSS/JS, 4. Review generated code, 5. Test in browser using Live Server, 6. Iteratively refine, 7. Optimize for mobile, 8. Deploy.
- Module 4 Project: Build a fully responsive company website with the following sections: Navigation bar with logo and menu links, Hero section with headline, subheading, and two CTA buttons, About Us section, Services section (6 cards), Team section, Testimonials section, and Contact form. The site must work perfectly on mobile and desktop. Use AI for all code generation but make at least 10 manual modifications to personalise it.

MODULE 5: NO-CODE AND LOW-CODE DEVELOPMENT
- Learning Objectives: Understand no-code and low-code platforms and their use cases, build apps on Bubble, FlutterFlow, and related platforms, create CRM and management systems visually, understand when to use no-code vs custom code.
- Platforms Overview:
  * Bubble (bubble.io) [Complex]: Full web app builder. Handles frontend, backend, and database in one place. Best for complex web apps, marketplaces, and SaaS.
  * FlutterFlow (flutterflow.io) [Medium]: Visual mobile app builder using Flutter framework. Generates real Dart code. Best for cross-platform iOS/Android apps.
  * Softr (softr.io) [Easy]: Build web apps and portals connected to Airtable or Google Sheets. Fastest way to turn data into web app.
  * Glide (glideapps.com) [Easy]: Turn Google Sheets or Excel data into mobile apps. Perfect for internal/small business apps.
  * Adalo (adalo.com) [Easy]: Drag-and-drop mobile/web app builder with built-in database. Good for simple data models.
  * Airtable (airtable.com) [Easy]: Spreadsheet-database hybrid. Connects to other tools, acts as data layer.
- When to Use No-Code vs Custom Code:
  * Tight deadline/MVP: No-Code preferred.
  * Unique complex logic: Custom Code preferred.
  * Non-technical founder: No-Code preferred.
  * Highly scalable product: Custom Code preferred (No-code is risky).
  * Internal business tool: No-Code preferred.
  * Full design control: Custom Code preferred.
  * Budget constraints: No-Code preferred.

MODULE 6: UI/UX DESIGN WITH AI
- Learning Objectives: Understand core UI design principles, apply UX fundamentals, create wireframes and prototypes, use Figma, leverage AI design tools.
- UI Design Principles:
  * Visual Hierarchy: Use size, color, position to guide eyes. Headlines are larger, CTAs are bold/colored.
  * Consistency: Use same fonts, colors, spacing, and component styles throughout.
  * Whitespace: Gives design room to breathe, improves readability.
  * Contrast: Ensure sufficient contrast (min 4.5:1 ratio for normal text).
  * Alignment: Align elements to a grid to avoid looking unprofessional.
  * Colour Theory: Max 3 colors (primary, secondary, accent) plus neutrals.
  * Typography: Choose max 2 fonts (headings vs body), consistent sizes, weights, heights.
- UX Fundamentals: User Research, User Personas, User Flows (sign up -> browse -> add to cart -> checkout), Information Architecture, Accessibility, Usability Testing.
- AI Design Tools: Figma, Canva, Uizard (UI from text/sketches), Galileo AI (production-ready Figma designs from prompts).
- Module 6 Project: Design a Banking App Interface in Figma covering: Splash/Login screen, Dashboard (balance, recent transactions, quick actions), Send Money flow (3 screens), and Transaction History screen. Apply all UI principles. Export as clickable prototype and share Figma link.

MODULE 8: DATABASES AND BACKEND DEVELOPMENT
- Learning Objectives: Understand what databases are, set up and use Firebase/Supabase, understand MySQL, implement user authentication and data storage, create/consume REST APIs, build membership management system.
- Database Types:
  * Relational (SQL): Stored in tables with rows and columns. Examples: MySQL, PostgreSQL. Best for structured data, complex queries.
  * NoSQL Document: Stored as JSON-like documents. Flexible schema. Examples: MongoDB, Firestore. Best for fast development.
  * Realtime Database: Syncs instantly across clients. Example: Firebase Realtime DB. Best for chat, live dashboards.
  * Hosted BaaS (Backend-as-a-Service): Auth, storage, database in one platform. Examples: Firebase, Supabase. Best for rapid development, no server management.
- Firebase: Google's platform. Features: Firestore, Realtime Database, Storage, Hosting, Cloud Functions, Analytics.
- Supabase: Open-source alternative built on PostgreSQL. Real relational SQL with BaaS convenience.
- Module 8 Project: Create a Membership Management System with features: User registration (name, email, phone, membership plan), Login/logout, Dashboard showing all members, ability to activate/deactivate members, and search. Use Firebase or Supabase as backend. Build frontend as web app.

MODULE 14: ADVANCED AI AGENTS & PROFESSIONAL VIBE CODING
- Learning Objectives: Understand AI agents and multi-agent systems, work with OpenAI, Claude, and Gemini APIs, implement RAG (Retrieval-Augmented Generation) systems, understand MCP (Model Context Protocol), build AI Business Assistant, understand startups.
- AI Agent: perceives environment, makes decisions, takes actions autonomously.
- RAG: Retrieval-Augmented Generation. 1. Collect docs, 2. Split into chunks (500 words), 3. Convert chunks into embeddings, 4. Store in vector DB, 5. Semantic similarity search, 6. Pass chunks as context to AI.
- MCP (Model Context Protocol): Standard developed by Anthropic allowing AI models to connect to external tools/data sources (GitHub, Google Drive, Slack, databases).
